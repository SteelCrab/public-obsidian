# DBMS개요
## 데이터베이스 
>  데이터의 집합
## DBMS (Database Management System)
> 데이터베이스를 관리/운영하는 역할
## DBMS 주요 특징 
* 데이터 무결성
> 데이터의 정확성, 일관성, 유효성이 유지되는 것으로 정확성이란 중복이나 누락이 없는 상태를 의미하고, 일관성은 원인과 결과의 의미가 연속적으로 보장되어 변하지 않는 상태를 의미한다. 
* 데이터 독립성
> 데이터베이스의 물리적 구조나 데이터의 저장 방식에 변화가 생겨도 응용 프로그램이나 사용자에게는 영향을 주지 않는 능력을 의미한다.

| 물리적 데이터 독립성 | 논리적 데이터 독립성 |
|---               |---               |
| 데이터의 물리적 저장 방식이나 저장 장치 등이 변경되어도, 응용 프로그램에 영향을 주지 않는 특성을 말한다.| 데이터베이스의 논리적 구조가 변경되어도 응용 프로그램에는 영향을 주지 않는 특성을 말한다.|
|예를 들어, 데이터가 저장되는 디스크의 섹터 크기나 트랙 크기가 변해도 응용 프로그램은 영향을 받지 않아야 함| 예를 들어, 테이블 구조가 바뀌거나, 새로운 테이블이 추가되거나, 테이블 사이의 관계가 변경되어도 응용프로그램은 수정할 필요가 없어야 한다. |

* 데이터 보안
>DBMS는 사용자 인증 및 액세스 제어와 같은 무단 액세스 및 수정으로부터 데이터를 보호하기 위한 도구를 제공한다. 
* 데이터 중복의 최소화
> 데이터베이스의 데이터를 통합해서 관리해서 자료 중복, 데이터 종속성 해결
* 데이터의 일관성
> 하나의 데이터를 변경했을 때, 방생할 수 있는 데이터의 불일치성 배제

## 스키마(중요)
> 데이터베이스에 저장되는 데이터 구조오아 제약조건을 정의한 것이다.
## 인스턴스(x)
> 정의된 스키마에 따라 데이터베이스에 실제로 저장된 값
## 테이블(중요)
> 테이블은 행과 열로 구성된 데이터 집합으로 하나의 표모양으로 하고 있다.
* 키값 안가져도 됨
## 릴레이션
> 관계 데이터 모델의 테이블을 릴레이션이라고 한다. 튜플과 어트리뷰트로 구성되어 있으며 아래의 특징을 가짐

## 튜플 (중요) - 릴레이션의 행
> 릴레이션의 행을 의미하며, 파일 구조에서의 행(row), 레코드, 튜플은 유사한 의미를 가진다.
## 차수
> 릴레이션의 `속성(열)`의 갯수.
## 어트리뷰트(중요)
> 릴레이션의 열을 의미하며 파일 구조에서의 필드(Field), 열(Column)과 거의 유사한 의미를 갖는다.
## 카디널리트(중요)
>  하나의 릴레이션에서 튜플(행, row)의 전체 갯수.
